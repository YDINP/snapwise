커피를 주문하고 카운터에서 멍하니 기다리시나요? 아니면 **진동벨을 받고 자리에 앉아** 다른 일을 하시나요? 비동기 프로그래밍이 바로 진동벨입니다. JavaScript는 두 가지 벨을 제공합니다.

<!-- step:story -->

옛날 카페에서는 주문하면 **카운터 앞에서 계속 기다려야** 했습니다. 다른 손님들도 줄을 서서 기다렸죠.

```javascript
// 옛날 방식 (콜백 지옥)
orderCoffee(function(coffee) {
  drinkCoffee(coffee, function() {
    orderDessert(function(dessert) {
      eatDessert(dessert, function() {
        console.log("끝!");
      });
    });
  });
});
```

중첩이 깊어질수록 코드가 **피라미드**처럼 쌓였습니다.

어느 날, 카페 주인이 **진동벨 시스템**을 도입했습니다. 주문하면 벨을 받고, 준비되면 울립니다. 손님들은 자리에 앉아서 다른 일을 할 수 있었죠.

```javascript
// Promise 방식 (진동벨)
orderCoffee()
  .then(coffee => drinkCoffee(coffee))
  .then(() => orderDessert())
  .then(dessert => eatDessert(dessert))
  .catch(error => console.log("주문 실패"));
```

훨씬 읽기 쉬워졌습니다. 하지만 체인이 길어지면 여전히 복잡했죠.

그래서 카페가 **주문표 시스템**을 만들었습니다. 주문표에 적힌 순서대로 자동으로 가져다줍니다. 손님은 그냥 기다리면 됩니다.

```javascript
// async/await 방식 (주문표)
async function enjoyMeal() {
  const coffee = await orderCoffee();
  await drinkCoffee(coffee);
  const dessert = await orderDessert();
  await eatDessert(dessert);
  console.log("만족!");
}
```

**마치 동기 코드처럼 읽히지만**, 비동기로 실행됩니다.

<!-- step:reveal -->

**Promise vs async/await 핵심 차이:**

**Promise 방식**
- `.then()` 체인으로 연결
- 콜백 지옥을 해결했지만 여전히 중첩 가능
- 에러 처리는 `.catch()`
- 여러 Promise를 `Promise.all()`로 병렬 실행

**async/await 방식**
- **동기 코드처럼 읽히지만** 비동기로 실행
- `try/catch`로 깔끔한 에러 처리
- 코드가 직관적이고 읽기 쉬움
- `await`는 반드시 `async` 함수 안에서만 사용

<!-- step:tip -->

**실무 패턴:**

**병렬 실행 (빠름):**
```javascript
// 두 API를 동시에 호출
const [users, posts] = await Promise.all([
  fetchUsers(),
  fetchPosts()
]);
```

**순차 실행 (느림):**
```javascript
// 하나씩 기다림
const users = await fetchUsers();
const posts = await fetchPosts(); // users 완료 후 시작
```

**에러 처리:**
```javascript
try {
  const data = await fetchData();
} catch (error) {
  console.error("데이터 가져오기 실패:", error);
}
```

**오늘부터 실천**: 새로운 비동기 코드는 무조건 `async/await`로 작성하세요. 기존 Promise 코드도 리팩토링하면 **가독성이 2배 올라갑니다**. 단, 여러 API를 병렬로 호출할 땐 `Promise.all()`을 꼭 써서 **속도를 최적화**하세요!
