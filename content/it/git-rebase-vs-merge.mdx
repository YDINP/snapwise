브랜치를 합칠 때 rebase와 merge 중 뭘 써야 할까요?

**Merge의 특징**
- 두 브랜치의 히스토리를 **그대로 보존**합니다
- 병합 커밋(merge commit)이 자동 생성됩니다
- 협업 시 안전하고 추적이 쉬워요

**Rebase의 특징**
- 커밋 히스토리를 **일직선으로 정리**합니다
- 깔끔한 로그를 원할 때 유용해요
- 이미 push한 브랜치에는 사용 금지!

**언제 뭘 쓸까?**
- 공개 브랜치 합칠 때 → **merge**
- 로컬에서 정리할 때 → **rebase**

> rebase는 "히스토리 재작성"이므로 혼자 작업하는 브랜치에서만 사용하세요.
