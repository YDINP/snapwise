웹 페이지를 만들 때, 레이아웃이 **전쟁터**처럼 느껴진 적 있나요? `float`를 쓰다가 무너지고, `position: absolute`로 억지로 배치하다가 모바일에서 깨지고. 그런데 Grid와 Flexbox를 알면 이 전쟁이 **체스 게임**으로 바뀝니다.

<!-- step:story -->

두 디자이너가 같은 페이지를 만들었습니다.

**A 디자이너**는 Flexbox만 사용했습니다. 네비게이션 바는 완벽했죠. 버튼들이 균등하게 배치되고, 반응형도 쉬웠습니다. 하지만 복잡한 그리드 레이아웃을 만들려니 지옥이었습니다. **중첩된 Flexbox 9단**을 만들어야 했고, 코드는 300줄이 넘었습니다.

**B 디자이너**는 Grid만 사용했습니다. 페이지 전체 레이아웃은 완벽했죠. 헤더, 사이드바, 메인, 푸터가 깔끔하게 나뉘었습니다. 하지만 네비게이션 바의 버튼들을 가운데 정렬하려니 복잡했습니다. Grid는 **1차원 정렬에는 과한** 도구였죠.

**C 디자이너**는 둘을 섞어 썼습니다. Grid로 페이지를 나누고, 각 영역 내부는 Flexbox로 정렬했죠. 코드는 절반으로 줄었고, 유지보수도 쉬웠습니다.

3개월 뒤, C 디자이너만 모바일/태블릿/데스크톱 대응을 하루 만에 끝냈습니다. 도구를 올바르게 선택했으니까요.

<!-- step:reveal -->

**Grid vs Flexbox 핵심 차이:**

**Flexbox (1차원)**
- **행 또는 열** 중 하나만 제어
- 컴포넌트 내부 정렬에 최적
- `justify-content`, `align-items`로 간단한 정렬

**Grid (2차원)**
- **행과 열** 동시 제어
- 페이지 전체 레이아웃에 최적
- `grid-template-columns`, `grid-template-rows`로 복잡한 구조 생성

**언제 뭘 쓸까?**

**Grid를 사용하세요:**
- 복잡한 2D 레이아웃 (대시보드, 갤러리)
- 페이지 전체 구조 (헤더, 사이드바, 메인, 푸터)

**Flexbox를 사용하세요:**
- 네비게이션 바, 버튼 그룹
- 카드 내부 정렬 (제목, 내용, 버튼)
- 1차원 나열 (리스트, 태그)

**실무 조합:**
Grid로 페이지를 나누고 → 각 영역 내부는 Flexbox로 정렬

**오늘부터 실천**: 새 레이아웃을 만들 때 "2차원 구조인가, 1차원 나열인가?"를 먼저 물어보세요. **Grid + Flexbox 조합**을 익히면 거의 모든 레이아웃 문제를 해결할 수 있습니다. `float`는 이제 역사책에만 남겨두세요!
