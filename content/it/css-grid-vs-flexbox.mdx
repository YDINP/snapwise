<!-- step:cinematic-hook -->
네비게이션 바 버튼 3개를
가운데 정렬하는 데
30분이 걸렸다.

<!-- step:scene -->
Grid를 썼다.
grid-template-columns, justify-items...
너무 복잡했다.

<!-- step:dialogue:designer -->
"그건 Flexbox가 나아."

<!-- step:dialogue:junior -->
"Grid랑 뭐가 달라요?"

<!-- step:dialogue:senior -->
"Flexbox는 1차원 나열,
Grid는 2차원 격자."

<!-- step:narration -->
Flexbox로 바꿨다.
`display: flex; justify-content: center;`
3초 만에 완성.

<!-- step:impact -->
**Flexbox** → 1차원 (행 또는 열)
**Grid** → 2차원 (행과 열 동시)

<!-- step:narration -->
페이지 전체 레이아웃은 Grid.
헤더, 사이드바, 메인, 푸터.

각 영역 내부는 Flexbox.
버튼, 카드, 태그 정렬.

<!-- step:dialogue:senior -->
"Grid로 큰 틀을 나누고,
Flexbox로 안을 채워."

<!-- step:dialogue:junior -->
"레이아웃 문제는 거의 이 둘로 해결되겠네요."

<!-- step:reveal-title -->
🎨 CSS Grid vs Flexbox 언제 쓸까?
Grid는 페이지 구조,
Flexbox는 컴포넌트 정렬.

<!-- step:outro -->
**레이아웃 시간 90% 단축.**
Grid + Flexbox 조합으로 모든 레이아웃 해결.
