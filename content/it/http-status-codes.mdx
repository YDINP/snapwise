<!-- step:hook -->

식당에서 주문하면 직원이 "네, 알겠습니다(200)", "죄송하지만 품절입니다(404)", "지금 주방이 바빠요(503)" 같은 답을 하죠. 웹 서버도 똑같습니다. 브라우저가 요청하면 **숫자 코드**로 상황을 알려줍니다.

<!-- step:story -->

어느 날, 브라우저가 서버라는 식당을 찾아갔습니다.

**브라우저**: "메인 페이지 주세요!"
**서버**: **200 OK** — "여기 있습니다."

브라우저가 로그인 없이 마이페이지를 주문했습니다.

**브라우저**: "내 정보 보여주세요!"
**서버**: **401 Unauthorized** — "손님, 먼저 신분증 확인해야 해요."

브라우저가 존재하지 않는 메뉴를 찾았습니다.

**브라우저**: "/비밀-메뉴 주세요!"
**서버**: **404 Not Found** — "그런 메뉴 없는데요?"

브라우저가 파일을 올리려 했는데, 서버가 다운됐습니다.

**브라우저**: "사진 저장해주세요!"
**서버**: **500 Internal Server Error** — "죄송합니다, 지금 주방 고장났어요."

브라우저가 캐시된 이미지를 확인했습니다.

**브라우저**: "이 이미지 바뀌었나요?"
**서버**: **304 Not Modified** — "그대로예요. 냉장고에 있는 거 드세요."

<!-- step:reveal -->

**HTTP 상태 코드 그룹별 정리:**

**1xx — 정보 응답**
> "요청 받았고, 지금 처리 중이에요."

**2xx — 성공**
- **200 OK** — 요청 성공 (가장 흔함)
- **201 Created** — 새 리소스 생성 완료 (POST 성공)
- **204 No Content** — 성공했지만 보낼 내용 없음 (DELETE 성공)

**3xx — 리다이렉션**
- **301 Moved Permanently** — 주소가 영구 이동했어요
- **302 Found** — 임시로 다른 곳으로 가세요
- **304 Not Modified** — 캐시 그대로 써도 됩니다

**4xx — 클라이언트 오류**
> "손님이 잘못 주문하셨어요."
- **400 Bad Request** — 요청 형식이 이상해요
- **401 Unauthorized** — 로그인 필요
- **403 Forbidden** — 권한 없음
- **404 Not Found** — 페이지 없음 (가장 유명)

**5xx — 서버 오류**
> "저희 서버가 문제예요."
- **500 Internal Server Error** — 서버 내부 오류
- **502 Bad Gateway** — 중간 서버 문제
- **503 Service Unavailable** — 서비스 이용 불가 (과부하/점검)

<!-- step:action -->

**오늘부터 실천**: API를 만들 때 **상황에 맞는 코드**를 반환하세요. 모든 에러를 `200`으로 보내지 말고, 생성은 `201`, 검증 실패는 `400`, 권한 없음은 `403`으로 구분하면 **클라이언트 디버깅이 10배 쉬워집니다**. 상태 코드는 서버와 클라이언트가 소통하는 공통 언어입니다!
