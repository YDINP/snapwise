웹 서버와 브라우저는 숫자로 대화합니다. `200`은 "성공!", `404`는 "없어요!", `500`은 "고장났어요!"를 뜻하죠. 이 **숫자 암호**를 모르면 에러 메시지를 영영 이해할 수 없습니다.

<!-- step:story -->

어느 날, 브라우저가 서버에게 웹페이지를 요청했습니다.

**브라우저**: "메인 페이지 주세요!"
**서버**: "200 — 여기 있습니다."

브라우저가 로그인 없이 마이페이지를 요청했습니다.

**브라우저**: "내 정보 보여주세요!"
**서버**: "401 — 먼저 로그인하세요."

브라우저가 존재하지 않는 페이지를 찾았습니다.

**브라우저**: "/secret-page 주세요!"
**서버**: "404 — 그런 페이지 없어요."

브라우저가 파일을 업로드하려 했는데, 서버가 다운됐습니다.

**브라우저**: "파일 저장해주세요!"
**서버**: "500 — 지금 고장났어요. 잠시만 기다려주세요."

브라우저가 캐시된 이미지를 확인했습니다.

**브라우저**: "이 이미지 변경됐나요?"
**서버**: "304 — 그대로예요. 캐시 쓰세요."

이렇게 서버와 브라우저는 **숫자 언어**로 대화하며, 수백만 명의 웹 경험을 만듭니다.

<!-- step:reveal -->

**HTTP 상태 코드 그룹별 정리:**

**1xx — 정보 응답**
- 요청을 받았고 처리 중

**2xx — 성공**
- **200 OK** — 요청 성공
- **201 Created** — 리소스 생성 완료
- **204 No Content** — 성공했지만 보낼 내용 없음

**3xx — 리다이렉션**
- **301 Moved Permanently** — 영구 이동
- **302 Found** — 임시 이동
- **304 Not Modified** — 캐시 사용 가능

**4xx — 클라이언트 오류**
- **400 Bad Request** — 잘못된 요청
- **401 Unauthorized** — 인증 필요
- **404 Not Found** — 리소스 없음

**5xx — 서버 오류**
- **500 Internal Server Error** — 서버 내부 오류
- **503 Service Unavailable** — 서비스 이용 불가

<!-- step:action -->

**오늘부터 실천**: API 응답에 적절한 상태 코드를 반환하세요. `200`만 쓰지 말고, `201`(생성), `204`(삭제), `400`(검증 실패)을 구분하면 **클라이언트 디버깅이 10배 쉬워집니다**. 상태 코드는 서버와 클라이언트가 소통하는 공통 언어입니다!
