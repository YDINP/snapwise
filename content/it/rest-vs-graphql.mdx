<!-- step:cinematic-hook -->
모바일 앱 메인 화면 하나를 그리는데
API를 **3번** 호출했다.
사용자들이 "느리다"며 이탈했다.

<!-- step:scene -->
GET /users/123
GET /posts?user=123
GET /comments?post=1

연속 네트워크 딜레이.
총 1.5초 로딩.

<!-- step:dialogue:senior -->
"한 번에 가져올 수 없어?"

<!-- step:dialogue:junior -->
"REST는 엔드포인트가 나뉘어 있잖아요."

<!-- step:dialogue:senior -->
"GraphQL 써봐.
필요한 데이터만 한 번에 요청할 수 있어."

<!-- step:narration -->
쿼리를 작성했다.
user, posts, comments를 한 번에.

<!-- step:impact -->
**query {
  user(id: 123) {
    name
    posts { title, comments { text } }
  }
}**

<!-- step:narration -->
네트워크 요청이 1/3로 줄었다.
앱이 빨라졌다.

<!-- step:dialogue:senior -->
"근데 캐싱은?
REST처럼 HTTP 캐시 못 써."

<!-- step:dialogue:junior -->
"Apollo Client로 직접 구현해야 하는구나."

<!-- step:narration -->
결국 하이브리드로 갔다.
간단한 CRUD는 REST,
복잡한 조합은 GraphQL.

<!-- step:reveal-title -->
🔌 REST vs GraphQL 한눈에 비교
REST는 뷔페(고정 메뉴),
GraphQL은 맞춤요리(원하는 것만).

<!-- step:outro -->
**네트워크 효율 3배 향상.**
복잡한 데이터는 GraphQL로, 간단한 건 REST로.
