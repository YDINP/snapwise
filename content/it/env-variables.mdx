<!-- step:cinematic-hook -->
첫 프로젝트.
API 키를 코드에 직접 넣고
`git push`를 눌렀다.

<!-- step:scene -->
15분 뒤.
GitHub 경고 이메일이 왔다.
"Exposed API Key Detected."

<!-- step:dialogue:senior -->
"환경변수 안 썼어?"

<!-- step:dialogue:junior -->
".env 파일 만들었는데요."

<!-- step:dialogue:senior -->
"그거 커밋했어?"

<!-- step:narration -->
`.gitignore`에 `.env`가 없었다.
API 키가 GitHub에 그대로 노출됐다.

<!-- step:impact -->
**AWS 콘솔에 EC2 인스턴스 50개**
해커 봇이 자동으로 키를 찾아냈다.
청구서: 10만 달러.

<!-- step:narration -->
즉시 키를 재발급하고
`.gitignore`에 `.env` 추가.
하지만 이미 히스토리에 남았다.

<!-- step:dialogue:senior -->
"환경변수는 코드가 아니라 비밀이야.
절대 커밋하지 마."

<!-- step:dialogue:junior -->
".env.example로 템플릿만 공유하는구나."

<!-- step:narration -->
규칙은 간단했다.
.env는 .gitignore에 추가.
배포는 Vercel 환경변수 설정으로.

<!-- step:reveal-title -->
🔐 환경변수 .env 제대로 쓰는 법
.env는 .gitignore 필수.
API 키 노출은 회사를 망친다.

<!-- step:outro -->
**보안 사고 100% 예방.**
.env 커밋 금지, .env.example로 템플릿 공유.
