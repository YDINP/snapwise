서버를 옮기는 건 마치 이사와 같습니다. 짐을 다 싸서 트럭에 싣고, 새 집에 내려서 풀어야 하죠. 그런데 만약 **짐을 통째로 복사**해서 순간이동 시킬 수 있다면? Docker가 바로 그 마법입니다.

<!-- step:story -->

옛날 서버실에는 "세팅의 마법사"라 불리는 개발자가 있었습니다. 새 서버를 띄우려면 그에게 부탁해야 했죠. Node.js 버전 맞추고, 환경변수 설정하고, 의존성 패키지 설치하는 데만 반나절이 걸렸습니다.

어느 날 인턴이 물었습니다. "왜 매번 똑같은 작업을 반복하나요?" 마법사가 답했죠. "서버마다 환경이 다르니까. 설정 하나만 틀려도 전체가 안 돌아가거든."

그 인턴은 고민 끝에 **컨테이너**를 도입했습니다. 앱과 설정을 박스에 통째로 담아서, 어떤 서버에서도 똑같이 실행되게 만든 거죠. 이제 배포는 박스 복사-붙여넣기처럼 간단해졌습니다.

마법사는 놀라워하며 물었습니다. "어떻게 한 거지?" 인턴이 웃으며 말했죠. "Docker 명령어 10개만 익히면 됩니다."

그날부터 서버실에서 밤샘 작업이 사라졌습니다. 배포는 클릭 한 번이면 끝났으니까요.

<!-- step:reveal -->

**Docker 필수 명령어 10개:**

1. **`docker run`** — 이미지를 컨테이너로 실행 (박스를 여는 것)
2. **`docker ps`** — 실행 중인 컨테이너 목록 확인
3. **`docker stop`** — 컨테이너 중지
4. **`docker rm`** — 컨테이너 삭제 (박스 버리기)
5. **`docker images`** — 로컬에 있는 이미지(박스) 목록 확인
6. **`docker pull`** — Docker Hub에서 이미지 다운로드
7. **`docker build`** — `Dockerfile`로 새 이미지 빌드 (박스 만들기)
8. **`docker exec`** — 실행 중인 컨테이너에서 명령어 실행
9. **`docker logs`** — 컨테이너의 로그 확인 (디버깅 필수)
10. **`docker-compose up`** — 여러 컨테이너를 한 번에 실행 (풀스택 앱용)

<!-- step:action -->

**오늘부터 실천**: 로컬 개발 환경을 컨테이너로 만들어보세요. `docker-compose.yml` 하나면 팀 전체가 똑같은 환경에서 작업할 수 있습니다. 이 10개 명령어만 익혀도 **Docker의 80%를 활용**할 수 있습니다!
